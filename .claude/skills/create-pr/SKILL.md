# PR 作成手順

## 1. 状態確認

以下を並列で実行する：

- `git status` で未追跡・未コミットの変更を確認
- `git log --oneline main..HEAD` で PR に含まれるコミット一覧を取得
- `git diff main...HEAD --stat` で変更ファイルの概要を取得
- `git diff main...HEAD` で差分の詳細を取得
- 現在のブランチがリモートにプッシュ済みか確認

未コミットの変更がある場合、PR作成前にコミットするかユーザーに確認する。

## 2. PR 内容の作成

差分とコミット履歴を分析し、以下のテンプレートに従って PR のタイトルと本文を作成する。

### タイトル規則

- Conventional Commits 形式に従う（例: `feat: ルーレット画面を追加`）
- 70文字以内に収める
- 日本語で記述する

### 本文テンプレート

ALWAYS use this exact template structure:

```
## 概要

<!-- 変更の目的と背景を1〜3文で説明 -->

Closes #<issue番号>

## 変更内容

| ファイル | 変更内容 |
|---|---|
| `path/to/file` | 変更の説明 |

## テスト計画

- [ ] テスト項目1
- [ ] テスト項目2
```

### テンプレート記入ルール

- **概要**: 変更の「なぜ」を書く。「何を」ではなく動機・背景を説明する
- **Closes #**: 関連する issue 番号がブランチ名から判別できる場合は必ず記載する（ブランチ名形式: `<issue番号>-<概要>`）。判別できない場合はユーザーに確認する
- **変更内容**: `git diff --stat` の結果をもとに、全変更ファイルを表にまとめる
- **テスト計画**: ユーザーが手動で検証できる具体的なチェック項目を書く

## 3. PR 作成

```bash
gh pr create --title "<タイトル>" --body "$(cat <<'EOF'
<本文>
EOF
)"
```

- プッシュが必要な場合は `git push -u origin <branch>` を先に実行する
- ベースブランチは `main` を使用する
