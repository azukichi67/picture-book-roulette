# 機能一覧・画面一覧・帳票一覧 生成スキル

## 出力する表の種類

要件定義書の内容に応じて、以下の表を生成する。帳票の記述がなければ帳票一覧は省略する。

1. 機能一覧
2. 画面一覧
3. 帳票一覧（帳票・レポート出力の記述がある場合のみ）

---

## 1. 機能一覧フォーマット

| 機能ID | 機能名 | 機能概要 | 優先度 | 関連画面 | 備考 |
|--------|--------|----------|--------|----------|------|

| 列 | 説明 | 形式・例 |
|----|------|---------|
| 機能ID | 一意の識別子 | `F-001`（連番） |
| 機能名 | 機能の名前（簡潔に） | `絵本登録` |
| 機能概要 | 何ができるかを**具体的・測定可能な言葉**で記述 | `カメラで表紙を撮影し、タイトルとともに登録する` |
| 優先度 | 必須 / 任意 | `必須` または `任意` |
| 関連画面 | どの画面で使うか | `登録画面（A）` |
| 備考 | 制約・補足（なければ空欄） | `登録本数0件の場合は非表示` |

## 作成ルール

1. **曖昧な言葉を使わない** — 「使いやすい」「早い」「たくさん」は禁止。数値や条件で表現する
   - NG: `素早く検索できる` → OK: `検索結果を2秒以内に表示する`
2. **要件を機能単位に分解する** — 1行1機能。複合機能は分割する
3. **非機能要件は含めない** — パフォーマンス・セキュリティ等は別表扱い
4. **スコープ外は含めない**
5. **IDは各表で独立した連番**（F-001、S-001、R-001）

## 手順

1. 要件定義書（または指定されたドキュメント）を読み込む
2. 機能要件・画面・帳票の記述を抽出する
3. 上記フォーマットに従い各表をMarkdownで順番に出力する（ドラフト）
4. 帳票の記述がなければ帳票一覧は省略する
5. **必ず** `requirements-reviewer` サブエージェントを呼び出し、ドラフトをレビューさせる
6. レビューで指摘された 🔴高・🟡中 の問題をすべて修正する
7. 修正済みの成果物を利用者に返す。末尾に「非機能要件・スコープ外は除外した」旨と、レビューで修正した主な点を一言添える

> **注意:** ステップ5を省略してはならない。レビューなしで成果物を返すことは禁止。
