# 画面モック作成スキル

## 概要

要件定義書・画面定義書を読み込み、指定された段階の画面モックを HTML + CSS で生成する。
出力は必ず `docs/mockups/` ディレクトリに保存する。

## ワークフロー

### Step 1: ドキュメントを読み込む

以下のファイルを読み込む（存在するものすべて）:
- `docs/requirements.md`
- `docs/screen-definitions.md`（または同等の画面定義書）
- `docs/concept.md`

### Step 2: モックの段階を確認する

ユーザーにどの段階で作成するかを確認する（まだ指定されていない場合）:

| 段階 | 名前 | 特徴 | 使う場面 |
|------|------|------|----------|
| 1 | ワイヤーフレーム | 線と四角だけ。色なし | 最初の方向性確認 |
| 2 | ローファイモック | 大まかなレイアウト。グレー中心 | 要件定義での合意形成 |
| 3 | ハイファイモック | 実際に近い見た目。色・画像あり | 最終確認・デザイン承認 |

### Step 3: 対象画面を確認する

全画面を作成するか、特定の画面のみかを確認する。
指定がなければ全画面を作成する。

### Step 4: HTML + CSS を生成する

各段階の実装指針は [references/mock-levels.md](references/mock-levels.md) を参照。

**必ず守ること:**

- 1ファイルに全画面を含める（画面遷移の流れが見えるよう縦並びまたはセクション分けで配置）
- 正常系・エラー時・データ0件時の状態を必ず含める
- ダミーデータは実際に近いものを使う（「テスト太郎」「aaaa」などは使わない）
- 全画面の流れを優先し、細部より全体構造を先に完成させる
- 外部ライブラリは使わず、HTML + CSS のみで実装する（Font Awesome などのCDN利用は可）
- レスポンシブ対応はターゲットデバイス（モバイル/PC）に合わせる

### Step 5: ファイルを保存する

`docs/mockups/` ディレクトリに保存する（ディレクトリがなければ作成する）:
- ファイル名: `{段階名}-{日付}.html`（例: `wireframe-20260326.html`、`lofi-20260326.html`）

作成後、ユーザーにファイルパスを伝え、ブラウザで開いて確認するよう案内する。
