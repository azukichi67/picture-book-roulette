# プロジェクト概要

**アプリ名:** えほんガチャ
**目的:** 就寝前の絵本選びをルーレットで楽しいイベントにする
**対象ユーザー:** 親子（就学前の子どもと保護者）

---

# ブランチ戦略（GitHub Flow）

- **メインブランチ:** `main`（常にデプロイ可能な状態を維持）
- **作業ブランチ:** `main` から作成し、命名は `<issue番号>-<概要>` 形式（例: `12-add-roulette`）
- issue 指定の実装は必ず git worktree 上で新ブランチを切って作業する
- 作業完了後は PR を作成し、レビュー後 `main` へマージする
- マージ後の作業ブランチは削除する

---

# コミットメッセージ規約

Conventional Commits（例: `feat: ○○`, `fix: ○○`, `chore: ○○`）
commitlint で自動検証される

---

# 行動原則

- 不明点は予測せず、かならずユーザーに確認すること
- 常にシンプルな解を選ぶこと
- 問題の根本を解決し、安易な回避策を取らないこと
- 変更前に関連コード・ファイルを確認し、影響範囲を最小化すること
- 既存コードの再利用を優先し、重複を生まないこと
- 存在しないファイル・情報を推測で語らないこと
- 3ステップ以上のタスクは必ずPlanモードで開始すること
- リサーチ・調査はサブエージェントに委譲すること
- サブエージェント1つにつき、タスクは1つに限定すること

---

# レビュー

- issue の実装作業が完了したら、必ず `code-reviewer` サブエージェントにコード差分をレビューさせること
- レビューで指摘された問題はすべて反映してから PR 作成・完了報告すること
