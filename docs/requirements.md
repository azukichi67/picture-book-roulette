# 要件定義書：えほんガチャ

**バージョン:** 1.1
**作成日:** 2026-03-25
**対象フェーズ:** Phase 1（基本機能）

---

## 1. エグゼクティブサマリー

「えほんガチャ」は、就寝前の絵本選びをルーレットで楽しいイベントに変えるWebアプリである。
親子で絵本を登録し、ルーレットで今夜の1冊を決めるというシンプルな3ステップ（登録・選ぶ・記録）に集中する。
Phase 1では1家族向けのローカルストレージベースのWebアプリとして、登録・ルーレット・図鑑の基本3機能を2026-04-25までにリリースする。

---

## 2. プロジェクト概要

### 2.1 目的

就寝前の「今日どの本にする？」という選択をルーレットに委ねることで、子どもが主体的に関われる楽しい就寝ルーティンを作る。

### 2.2 背景・課題

- 毎晩どの絵本にするか決める時間がかかる
- 子どもが自分で「決めた感」を持てるプロセスがない
- 家にある絵本の記録・管理が煩雑

### 2.3 スコープ

**Phase 1（本要件定義の対象）**
- 絵本の登録（写真＋タイトル手入力）
- ルーレット（登録した全絵本からランダム選出）
- 図鑑（タイル表示＋最終既読日の表示）
- データ保存：ブラウザのローカルストレージ

**Phase 2以降（スコープ外）**
- バーコードスキャンによる自動書誌情報取得
- クラウド同期（複数端末対応）
- プレイリスト・絞り込み機能
- 絵本情報の編集（Phase 1 では削除・再登録で対応）

## 3. ステークホルダー一覧

| 役割 | 説明 |
|---|---|
| 保護者（主操作者） | 絵本の登録・アプリの設定を行う |
| 子ども（就学前） | ルーレットのストップボタンを押す・図鑑を眺める |

※ 1家族専用アプリ。アカウント管理・マルチユーザー機能は対象外。

---

## 4. 業務フロー

### 4.1 As-Is（現状）

1. 就寝前に「今日どの本にする？」と親子で相談
2. 棚から本を選ぶ（時間がかかる・マンネリ化しやすい）
3. 読み終わり、記録は特になし

### 4.2 To-Be（導入後）

1. アプリを起動し、ルーレット画面を開く
2. 子どもがストップボタンを押して今夜の1冊を決める
3. 選ばれた絵本の表紙が全画面表示され、「決定」ボタンと「もう一度」ボタンが表示される
4. 「決定」を押すと、その時点でその絵本の既読日が当日の日付に記録される
5. 読み終わり後、図鑑に最終既読日が反映されている

---

## 5. 機能要件

### 5.1 機能一覧表

| 機能ID | 機能名 | 概要 | 優先度 |
|---|---|---|---|
| F-01 | 絵本登録 | 写真とタイトルを入力して絵本を登録する | 必須 |
| F-02 | ルーレット | 登録済み全絵本からランダムに1冊を選出する | 必須 |
| F-03 | 図鑑（一覧） | 登録済み絵本をタイル表示する | 必須 |
| F-04 | 既読記録 | 決定画面で「決定」を押した時点で最終既読日を記録する | 必須 |
| F-05 | 絵本削除 | 登録済み絵本をリストから削除する | 必須 |

### 5.2 機能詳細

#### F-01: 絵本登録

- 保護者はカメラで絵本の表紙を撮影、またはカメラロールから画像を選択できる（必須）
- 保護者はタイトルを手入力できる（必須・1〜30文字・空白のみ無効）
- 登録完了ボタン押下後、0.5〜1.0秒のアニメーション演出（図鑑にシュッと吸い込まれる）を表示する
- アニメーション完了後、図鑑画面（S-03）に遷移する
- 登録された絵本はローカルストレージに保存される

**例外:**
- タイトルが未入力または空白のみの場合、「タイトルを入力してください」と表示し登録を拒否する
- 画像が未選択の場合、「表紙の写真を選んでください」と表示し登録を拒否する
- ローカルストレージの空き容量が不足している場合、「データを保存できませんでした。不要な絵本を削除してください」と表示する

#### F-02: ルーレット

- 登録済み全絵本の表紙が縦にスロット形式で高速回転するアニメーションを表示する（初速: 最大20枚/秒相当）
- 子どもが押しやすい大きなストップボタン（最小タップ領域 60×60px）を画面中央付近に表示する
- ストップボタン押下後、1秒以内にアニメーションが停止し、決定画面（S-04）に遷移する

**例外:**
- 登録絵本が0冊の場合、「まず絵本を登録してね」と表示し、登録画面（S-02）へ誘導する
- 登録絵本が1冊の場合、短めのアニメーション演出（同じ1冊が数回回る）を挟んでから決定画面（S-04）に遷移する

#### F-03: 図鑑（一覧）

- 登録済み絵本の表紙をタイル形式（グリッドレイアウト）で一覧表示する
- 各タイルに表紙画像（最小80×80px）とタイトルを表示する
- 既読履歴がある絵本には最終既読日を控えめに表示する（例：「3/20」）
- 未読の絵本には日付を表示しない

#### F-04: 既読記録（手動確認）

- 決定画面（S-04）で保護者が「決定」ボタンを押した時点で、その絵本の最終既読日を当日の日付（YYYY-MM-DD形式）に更新する
- 「もう一度」ボタンを押した場合は既読日を記録せず、ルーレット画面（S-01）に戻る
- 記録するのは「最後に決定した日」1件のみ（履歴は保持しない）

#### F-05: 絵本削除

- 図鑑画面（S-03）から絵本を削除できる
- 削除前に確認ダイアログを表示する（「この絵本を削除しますか？」）
- 削除すると絵本データ（画像・タイトル・既読日）がローカルストレージから削除される

**例外:**
- 確認ダイアログで「キャンセル」を押した場合、図鑑画面（S-03）に留まり削除しない

### 5.3 画面一覧

| 画面ID | 画面名 | 概要 |
|---|---|---|
| S-01 | ルーレット画面 | アプリ起動時のホーム。ルーレットを実行する |
| S-02 | 絵本登録画面 | 写真とタイトルを入力して絵本を登録する |
| S-03 | 図鑑画面 | 登録済み絵本をタイル表示する |
| S-04 | 決定画面 | ルーレット結果（選ばれた絵本）を全画面表示する |

### 5.4 画面遷移

| 画面 | 操作 | 遷移先 |
|---|---|---|
| S-01（ルーレット） | ストップボタン押下 | S-04（決定画面） |
| S-01（ルーレット） | 「登録する」ボタン押下 | S-02（絵本登録） |
| S-01（ルーレット） | 「図鑑」ボタン押下 | S-03（図鑑） |
| S-02（絵本登録） | 登録完了アニメーション終了 | S-03（図鑑） |
| S-02（絵本登録） | 「戻る」押下 | 遷移元画面（S-01 または S-03） |
| S-03（図鑑） | 「戻る」押下 | S-01（ルーレット） |
| S-04（決定画面） | 「決定」ボタン押下 | S-01（ルーレット） |
| S-04（決定画面） | 「もう一度」ボタン押下 | S-01（ルーレット） |

## 6. データモデル

### 絵本エンティティ（Book）

| フィールド名 | 型 | 必須 | 説明 |
|---|---|---|---|
| `id` | string（UUID） | 必須 | 絵本を一意に識別するID |
| `title` | string | 必須 | 絵本のタイトル（1〜30文字） |
| `imageData` | string（Base64） | 必須 | 表紙画像（JPEG/PNG をBase64エンコード） |
| `lastReadDate` | string（YYYY-MM-DD）または null | 任意 | 最後に「決定」した日付。未読の場合は null |
| `createdAt` | string（YYYY-MM-DD） | 必須 | 登録日 |

**保存形式:** JSON配列としてローカルストレージの1キーに保存する
**注意:** 画像はアプリ側で長辺500pxにリサイズしてからBase64で保存する（1枚あたり約50〜100KB）。登録冊数が増えるとストレージ容量（通常5〜10MB）を圧迫する可能性があるため、容量不足時はエラーハンドリングで対応する

## 7. 非機能要件

### 7.1 可用性

- オフライン環境でも動作すること（ローカルストレージのみ使用のため）
- ブラウザを閉じてもデータが消えないこと（ローカルストレージに永続化）

### 7.2 性能

以下はすべて「ローカルストレージからの読み込み」を前提とした基準とする（ネットワーク速度は対象外）。測定対象デバイスはミドルレンジのスマートフォン（iPhone SE 第3世代相当）とする。

- ルーレット画面の表示は起動後3秒以内に完了すること
- ストップボタン押下からアニメーション停止まで1秒以内に応答すること
- 登録絵本が100冊あっても図鑑画面が3秒以内に表示されること

### 7.3 ユーザビリティ

- 主操作対象はスマートフォンのブラウザ（Safari / Chrome）とする
- ストップボタンは子どもの指で押しやすいよう最小タップ領域を60×60px以上とする
- 図鑑画面の各タイルは表紙画像を最小80×80px以上で表示すること

### 7.4 拡張性

- Phase 2でのクラウド同期を想定し、データ構造はJSONシリアライズ可能な形式で設計すること
- バーコードスキャン機能を後から追加できるよう登録フローを疎結合に設計すること

### 7.5 運用・保守

- 初回起動時に「データはこのブラウザにのみ保存されます。ブラウザのデータ消去で失われます」というメッセージをモーダルで1回だけ表示する
- 外部サービス依存がないため、サーバー保守は不要

### 7.6 セキュリティ

- 個人情報（写真・タイトル）はローカルストレージにのみ保存し、外部送信しないこと
- 入力値はXSSを防ぐためサニタイズして表示すること

---

## 8. 制約条件・前提条件

| 区分 | 内容 |
|---|---|
| 制約 | データはブラウザのローカルストレージに保存するため、ブラウザのデータ消去でデータが失われる |
| 制約 | 1台のデバイス・1ブラウザのみで使用することを前提とする |
| 制約 | 絵本情報（タイトル・写真）の編集機能はPhase 1では提供しない。変更が必要な場合は削除して再登録する |
| 前提 | ユーザーはスマートフォンを所持し、カメラ機能を利用できる |
| 前提 | ネットワーク接続は初回読み込み時のみ必要（以降はオフライン可） |
